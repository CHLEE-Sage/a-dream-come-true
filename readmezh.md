# 私人財務顧問 Agent

一個以隱私優先、本地優先為基礎打造的財務規劃 Agent。  
它的設計目標，不是做成被動式計算機，而是更像一位會先理解客戶需求、再開始診斷的專業財務顧問。

## 專案路徑
- `C:\Agentgroups\workspaces\private-finance-agent`

## 核心概念
這個產品不是單純的退休試算表，也不是只有輸入欄位的表單工具。

它希望做到的是：

- 先理解使用者在意的目標與夢想
- 協助發現尚未被納入規劃的風險與人生事件
- 看見未來某一年真正危險的現金流壓力
- 將建議轉成具體事件與時間軸
- 在保護隱私的前提下，提供像顧問一樣的初步診斷

簡單說：

> 這個產品想成為一位會先聽懂你，再開始談錢的私人財務顧問。

## 產品藍圖

### 1. 隱私優先
原則：

- 預設本地優先
- 不強制登入
- 不要求真實姓名、電話或帳號識別資訊
- 可匯出資料
- 可由使用者自行重設或刪除

未來方向：

- 前端加密
- 更強的本地資料保護
- 若真的需要，再考慮匿名雲端同步

### 2. 顧問式探索
產品不該等使用者自己想到所有財務項目。
它應該像專業顧問一樣，主動追問真正重要的事。

探索範圍包含：

- 家庭結構
- 孩子與教育
- 成家與家庭支持
- 房屋、房貸、搬家、裝修
- 車輛購置與更換
- 退休時間
- 年金開始時間
- 保險缺口
- 父母照護與長照
- 房產出售或資產變現
- 退休後兼職或副收入

### 3. 事件驅動規劃引擎
這個產品的核心不是只看某一刻資產，而是用事件去理解人生中的財務壓力。

它要能建模：

- 大額未來支出
- 未來收入變化
- 保障補強
- 資產出售事件
- 退休前後的過渡壓力

常見事件包含：

- 教育支出高峰
- 成家支持
- 買車 / 換車
- 搬家 / 裝修
- 房產出售 / 以小換大或以大換小
- 保險補強
- 父母照護成本
- 退休後副收入

### 4. 缺口排序邏輯
產品不能只是指出問題，還必須幫使用者排序優先順序。

一個實用的排序方式：

#### Level 1 - 關鍵
- 流動性不足
- 負債壓力過高
- 65 歲前退休銜接風險
- 重大保障缺口

#### Level 2 - 重要
- 退休資金缺口
- 儲蓄率偏低
- 中期大額支出沒有正式規劃
- 家庭支持壓力

#### Level 3 - 優化
- 資產配置效率
- 稅務 / 傳承優化
- 時點微調

### 5. 視覺化規劃體驗
介面應該像規劃師的工作台，而不是冰冷試算表。

核心視圖包含：

- 財務健康概覽
- 流動資產時間軸
- 年度現金流推估
- 關鍵事件規劃
- 情境比較
- 顧問摘要
- 缺口優先排序

### 6. AI 顧問層
產品可接 OpenAI API，利用你自訂的 instruction 與 knowledge 產生第一輪 AI 財務診斷。

目前架構已支援：

- 後端 `/api/diagnosis`
- 自訂顧問指令檔
- 自訂知識檔
- 卡片式 AI 診斷輸出

## 本地開發
```bash
npm install
npm run dev
```

開啟：
- `http://127.0.0.1:3000`

若要啟用 AI 診斷：

1. 建立 `.env.local`
2. 設定：

```env
OPENAI_API_KEY=你的金鑰
OPENAI_MODEL=gpt-5-mini
```

3. 編輯：
- `src/prompts/advisor-instructions.md`
- `src/prompts/advisor-knowledge.md`
